# 系统架构设计师备考资料体系 - 设计文档

## 背景

用户正在备考2026年5月的软考系统架构设计师考试，已有两周刷题经验但缺乏系统知识框架，有一定开发经验但架构经验偏少。需要在约1个月内构建完整的知识体系，以md格式输出，使用Obsidian管理，后续同步至知识网站。

## 设计策略

**核心理念：先建框架，再用框架串题**

用户已刷题两周但效率不高，根因是脑海中没有结构化的知识框架。本资料体系优先构建思维导图和知识全景图，让每个知识点都有"位置感"，然后通过标签和双向链接将刷题经验与框架挂钩。

**优先级策略（按红宝书重点级别对齐）**：

- P0（第1-2周）：超级重点★★★★★★ — 系统架构设计(ch4)、需求工程(ch2)、系统设计(ch3)、数据库(ch12) + 案例/论文解题框架
- P1（第2-3周）：重点★★★★★ — 软件工程概述(ch1)、法律法规(ch14) + 案例分析各专题
- P2（第3-4周）：次重点★★★☆ — 计算机组成(ch9)、操作系统(ch11)、网络(ch10)、测试(ch5)、运维(ch6)、可靠性(ch7)、项目管理(ch8)、信息安全(ch13) + 论文素材库
- P3（考前3天）：非重点 — 信息系统/未来技术/数学/英语 合并精简 + 速查手册集中复习

## 单文件内部结构规范

每个知识点 md 文件遵循统一结构：

```markdown
# 章节标题

## 知识全景（mermaid 思维导图）

用 mermaid mindmap 呈现本章知识框架，建立"位置感"。

## 各节知识点

### X.Y 小节标题 #高频/#中频/#低频

**核心概念**：一句话定义

**要点归纳**：
- 关键点1
- 关键点2

**易错点/对比辨析**（如有）：#易错
| 概念A | 概念B |
|-------|-------|
| ...   | ...   |

**与其他知识点的关联**：
- 相关案例分析：[[02-案例分析/XX]]
- 相关论文方向：[[03-论文/XX]]
```

## Obsidian 适配规范

- **双向链接**：`[[文件名]]` 或 `[[文件名#小节]]` 互相引用
- **标签体系**：
  - 频率：`#高频` `#中频` `#低频`
  - 特殊标记：`#易错` `#必背` `#公式`
  - 科目：`#综合知识` `#案例分析` `#论文`
- **MOC.md**：作为 Obsidian Graph View 的中心节点，链接所有模块
- **mermaid 图表**：Obsidian 原生支持，无需插件

## 内容编写原则

1. **框架优先**：每章先给思维导图，再展开细节
2. **考试导向**：高频知识点详细展开，低频知识点精简概括
3. **对比记忆**：易混淆概念用表格对比（如SAAM vs ATAM、DES vs AES vs RSA）
4. **链接串联**：综合知识、案例分析、论文三科之间通过双向链接互相引用
5. **实战挂钩**：案例分析和论文部分提供答题模板和项目素材
